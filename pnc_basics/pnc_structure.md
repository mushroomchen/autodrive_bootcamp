PNC 架构说明（简化版）

我当前构建的自动驾驶 PNC pipeline 采用分层式结构，主要包括 Behavior、Path、Trajectory 和 Control 四个层次。该架构的设计思路是将“行为决策—路径生成—轨迹时标化—底层控制执行”这几个功能模块进行解耦，以降低系统复杂度，并使各层任务边界更加清晰。

在最上层，Behavior 层根据当前交通场景和车辆运行状态决定车辆的行为模式，例如 keep lane、follow、lane change 和 stop。该层不直接生成具体轨迹，而是输出当前阶段的驾驶意图，为下层规划提供模式约束。

Path 层根据 Behavior 层的输出结果以及道路几何信息，生成一条局部几何参考路径。该层主要解决“往哪里走”的问题，重点关注路径的空间形状和平滑性，而不严格规定车辆在各时刻到达路径上哪个位置。对于简化场景，可采用车道中心线提取、样条曲线或多项式曲线方式生成局部路径。

Trajectory 层在参考路径基础上进一步引入时间参数，生成可执行的时标化轨迹。该层需要综合考虑车辆当前速度、加速度、曲率约束、障碍物约束以及舒适性要求，从而得到一组满足动力学和安全要求的轨迹候选，并从中选择最优轨迹。因此，Trajectory 层主要解决“以什么速度、按什么节奏走”的问题。

Control 层负责跟踪上层给定的参考轨迹，并输出实际控制命令，如转向、油门和制动。该层直接与车辆动力学和执行器相连，是规划结果落地到车辆行为的关键环节。在当前简化实现中，横向控制器可采用 Pure Pursuit、Stanley 或 LQR 等方法，纵向控制器可采用 PID 或简化 ACC 方法。

整体而言，该 PNC 架构从高层行为决策到低层控制执行形成了清晰的层次链路。其中，Behavior 决定“做什么”，Path 决定“往哪里走”，Trajectory 决定“怎么在时间上走”，Control 则决定“如何将规划结果转化为车辆控制输入”。这种分层结构有助于逐步搭建和验证一个简化但完整的自动驾驶规划与控制系统。
